# Feature Specification: Premium Cleaning Service Website

**Feature Branch**: `001-premium-cleaning-website`  
**Created**: December 4, 2025  
**Status**: Draft  
**Input**: User description: "프리미엄 공실 청소 서비스 기업의 웹사이트 전체 구조, 콘텐츠, 카피라이팅, 디자인 가이드를 구성한다. 블랙 베이스 + 블루 포인트 컬러, 모던한 톤&무드, 마블 질감을 활용한 고급 디자인"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Prospect Explores Service Overview (Priority: P1)

신규 고객이 웹사이트에 방문하여 회사의 서비스와 브랜드를 빠르게 이해하고 주요 가치를 파악한다.

**Why this priority**: 모든 고객의 첫 경험이며, 웹사이트의 가장 중요한 진입점. 이 단계에서 브랜드 인상과 신뢰도가 결정되므로 최우선 순위.

**Independent Test**: 홈페이지에서 히어로 섹션, 브랜드 철학, 핵심 서비스 3개를 확인하면, 사용자가 "무엇을 하는 회사인지" "어떤 가치를 제공하는지" 명확히 이해할 수 있어야 함.

**Acceptance Scenarios**:

1. **Given** 사용자가 웹사이트에 진입, **When** 홈페이지 로드, **Then** 메인 카피 "하루 한 집, 깊이를 청소합니다"와 서브 카피가 명확하게 표시되고, 시각적으로 고급스럽고 단정한 느낌(블랙 베이스 + 블루 포인트, 마블 질감)이 전달됨
2. **Given** 사용자가 홈페이지를 스크롤, **When** 브랜드 철학 섹션 도달, **Then** 6개 철학 항목(하루 한 집 집중, 마음과 책임, 전문 장비 사용 등)이 아이콘과 함께 명확하게 표시됨
3. **Given** 사용자가 계속 스크롤, **When** 핵심 서비스 카드 섹션 도달, **Then** 입주청소, 이사청소, 인테리어 후 청소 3개 서비스가 각각 설명과 "자세히 보기" 버튼과 함께 표시됨

---

### User Story 2 - Prospect Requests Quote/Reservation (Priority: P1)

관심 있는 고객이 간편하게 견적/예약을 요청한다.

**Why this priority**: 비즈니스의 핵심 전환점(conversion). 사용자가 서비스에 관심을 보일 때 즉시 예약할 수 있는 경로가 필수적이므로 P1.

**Independent Test**: 어느 페이지에서든 "견적 / 예약 요청하기" 버튼을 클릭하면 구글폼 링크로 이동하여 예약 요청 폼을 작성할 수 있어야 함.

**Acceptance Scenarios**:

1. **Given** 사용자가 홈페이지의 히어로 섹션, **When** "간편견적문의" 버튼 클릭, **Then** 구글폼 링크로 이동
2. **Given** 사용자가 다른 페이지(서비스, 사례 등), **When** 각 페이지의 "견적 / 예약 요청하기" 또는 "자세히 보기" 버튼 클릭, **Then** 구글폼 링크로 이동
3. **Given** 사용자가 모바일 디바이스 사용, **When** 네비게이션 메뉴에서 "견적 / 예약" 탭 클릭, **Then** 구글폼 링크로 이동

---

### User Story 3 - Prospect Learns About Service Details (Priority: P1)

고객이 특정 서비스(입주청소, 이사청소, 인테리어 후 청소)의 상세 정보를 조회한다.

**Why this priority**: 고객이 자신의 상황에 맞는 서비스를 이해하고 결정하는 데 필수적. 각 서비스의 범위, 필요 상황, 기본 청소 범위 등을 명확히 해야 하므로 P1.

**Independent Test**: 서비스(Service) 페이지에서 3개 서비스의 상세 설명, 기본 청소 범위(현관, 방, 창문, 주방, 욕실, 베란다, 수납공간 등), 옵션 안내가 명확하게 표시되고, 각 서비스가 어떤 상황에서 필요한지 이해할 수 있어야 함.

**Acceptance Scenarios**:

1. **Given** 사용자가 서비스 페이지 접속, **When** 페이지 로드, **Then** 3개 서비스 카드가 표시되고, 각 서비스의 설명과 "자세히 보기" 버튼이 있음
2. **Given** 사용자가 특정 서비스 카드의 "자세히 보기" 클릭, **When** 상세 페이지/섹션 전개, **Then** 서비스 설명, 기본 청소 범위(7개 영역), 제한 사항(외창 제외 등), 옵션 안내가 명확하게 표시됨
3. **Given** 사용자가 상세 페이지 하단, **When** 스크롤, **Then** "견적 / 예약 요청하기" 버튼이 표시됨

---

### User Story 4 - Prospect Understands How Work Process (Priority: P2)

고객이 실제 작업이 어떻게 진행되는지, 투명성 있게 이해한다.

**Why this priority**: 브랜드 신뢰도와 차별성(실시간 촬영 공유, 전담제 등)을 전달하는 중요한 요소이지만, 예약 후 알아도 되므로 P2.

**Independent Test**: "How We Work" 페이지에서 9단계 프로세스(상담 → 확정 → 현장 점검 → 장비 셋업 → 영역별 청소 → 문제 구간 처리 → 실시간 촬영 → 검수 → 결과 안내)와 주요 강점(하루 한 집 전담, 전문 장비, 실시간 공유)이 명확하게 표시되어야 함.

**Acceptance Scenarios**:

1. **Given** 사용자가 "How We Work" 페이지 접속, **When** 페이지 로드, **Then** 9단계 프로세스가 시각적으로 표시됨(예: 타임라인, 카드 형식 등)
2. **Given** 사용자가 페이지를 스크롤, **When** 각 단계에 도달, **Then** 단계별 설명과 해당 활동이 명확함
3. **Given** 사용자가 하단 도달, **When** 확인, **Then** "고객 부재 시에도 촬영 결과를 통해 전 과정을 확인 가능합니다" 문구가 표시됨

---

### User Story 5 - Prospect Learns About Equipment & System (Priority: P2)

고객이 회사가 사용하는 전문 장비와 시스템을 이해하고, 안심감을 느낀다.

**Why this priority**: 차별성과 신뢰도 강화에 중요하지만, 예약 후에도 알 수 있으므로 P2.

**Independent Test**: "Equipment" 페이지에서 4개 장비(전문 흡입 시스템, 고온 스팀 시스템, 바닥 폴리싱 시스템, 친환경 약품)와 각 장비의 용도 및 효과(분진 제거, 냄새 완화, 위생 개선)가 명확하게 표시되어야 함.

**Acceptance Scenarios**:

1. **Given** 사용자가 "Equipment" 페이지 접속, **When** 페이지 로드, **Then** 4개 장비가 카드 형식으로 표시됨
2. **Given** 각 장비 카드, **When** 확인, **Then** 장비명, 설명, 용도가 명확함
3. **Given** 사용자가 페이지 하단, **When** 도달, **Then** "분진 제거, 냄새 완화, 위생 개선까지 입주 가능한 기본 컨디션을 완성합니다" 요약 문구가 표시됨

---

### User Story 6 - Prospect Reviews Case Studies & Customer Feedback (Priority: P2)

고객이 실제 시공 사례와 고객 후기를 통해 신뢰도를 확인한다.

**Why this priority**: 사회적 증거(social proof)로 신뢰도를 높이는 요소이지만, 핵심 의사결정 이후에도 영향을 미치므로 P2.

**Independent Test**: "Case Study" 페이지에서 최소 3개 이상의 사례가 표시되고, 각 사례에 작업 유형(입주/이사/인테리어), 평수, 현장 문제점, 사용 장비, 실제 작업 방식, 결과, 고객 피드백이 포함되어 있어야 함. "Review" 페이지에서 고객 후기 6개 이상의 강조 문구(차이 확실, 장비 수준 높음, 완성도 높음 등)가 표시되어야 함.

**Acceptance Scenarios**:

1. **Given** 사용자가 "Case Study" 페이지 접속, **When** 페이지 로드, **Then** 사례 카드들이 그리드 형식으로 표시됨
2. **Given** 사용자가 특정 사례 카드 클릭, **When** 상세 보기, **Then** 작업 유형, 평수, 문제점, 장비, 방식, 결과, 고객 피드백이 모두 표시됨
3. **Given** 사용자가 "Review" 페이지 접속, **When** 페이지 로드, **Then** 고객 후기 스크린샷 또는 텍스트 요약이 표시되고, 6개의 강조 문구가 시각적으로 돋보임

---

### User Story 7 - Prospect Reads Terms & Conditions Before Booking (Priority: P2)

고객이 제한 사항, 추가 비용, 기본 준비 사항, A/S 정책, 취소/위약금을 미리 확인한다.

**Why this priority**: 고객 분쟁 방지와 사업 운영의 명확성을 위해 중요하지만, 예약 단계에서 확인할 수 있으므로 P2.

**Independent Test**: "Notice" 페이지에서 4개 섹션(작업 제한 및 안내, 추가 비용 발생 기준, 기본 준비 사항, A/S 정책, 취소·위약금)과 각 항목들이 명확하게 표시되어야 함.

**Acceptance Scenarios**:

1. **Given** 사용자가 "Notice" 페이지 접속, **When** 페이지 로드, **Then** 5개 섹션이 아코디언 또는 탭 형식으로 표시됨
2. **Given** 사용자가 각 섹션 클릭, **When** 상세 내용 전개, **Then** 각 섹션의 항목들이 명확하게 나열됨
3. **Given** 사용자가 "취소·위약금" 섹션 확인, **When** 내용 검토, **Then** 일정별 환불 비율(7일 전 100%, 6-4일 80% 등)과 현장 작업 불가 기준이 명확함

---

### User Story 8 - Prospect Navigates Website on Mobile (Priority: P2)

모바일 사용자가 햄버거 메뉴를 통해 쉽게 네비게이션할 수 있다.

**Why this priority**: 모바일 접근성은 필수이지만, 모바일 전환 이후의 과제이므로 P2.

**Independent Test**: 모바일 뷰에서 화면 상단에 햄버거 메뉴 아이콘이 표시되고, 클릭 시 전체 메뉴(Home, 소개, 서비스, 작업 방식, 장비 & 시스템, 시공 사례, 후기, 이용 안내, 견적 / 예약)가 슬라이드 또는 모달로 표시되어야 함.

**Acceptance Scenarios**:

1. **Given** 사용자가 모바일 디바이스에서 웹사이트 접속, **When** 페이지 로드, **Then** 화면 상단에 햄버거 메뉴 아이콘이 표시됨
2. **Given** 사용자가 햄버거 메뉴 클릭, **When** 메뉴 전개, **Then** 9개의 모든 메뉴 항목이 표시됨
3. **Given** 사용자가 메뉴에서 특정 항목 클릭, **When** 이동, **Then** 해당 페이지로 정상 이동하고, 메뉴가 닫힘

---

### User Story 9 - User Connects to Social Media (Priority: P3)

사용자가 홈페이지에서 유튜브와 인스타그램으로 쉽게 이동한다.

**Why this priority**: 소셜 미디어 연결은 좋은 부가 기능이지만, 예약 이후의 고객 유지 전략이므로 P3.

**Independent Test**: 홈페이지의 히어로 섹션(또는 푸터)에 유튜브와 인스타그램 버튼이 표시되고, 각각 클릭 시 해당 소셜 미디어 링크로 이동해야 함.

**Acceptance Scenarios**:

1. **Given** 사용자가 홈페이지 히어로 섹션, **When** 확인, **Then** 유튜브 버튼과 인스타그램 버튼이 표시됨
2. **Given** 사용자가 유튜브 버튼 클릭, **When** 클릭, **Then** 회사 유튜브 채널로 이동
3. **Given** 사용자가 인스타그램 버튼 클릭, **When** 클릭, **Then** 회사 인스타그램 계정으로 이동

---

### User Story 10 - Administrator Manages Website Content (Priority: P3)

관리자가 웹사이트 콘텐츠(서비스, 사례, 후기 등)를 쉽게 관리할 수 있다.

**Why this priority**: 시간이 지남에 따라 콘텐츠를 업데이트해야 하므로 중요하지만, 초기 론칭 이후의 과제이므로 P3.

**Independent Test**: 관리 패널이 제공되거나, 콘텐츠 관리 시스템이 구축되어, 최소한 사례와 후기 정보를 쉽게 추가/수정할 수 있어야 함.

**Acceptance Scenarios**:

1. **Given** 관리자가 관리 페이지 접속, **When** 인증, **Then** 관리 대시보드가 표시됨
2. **Given** 관리자가 "사례" 관리 섹션 클릭, **When** 이동, **Then** 기존 사례 목록이 표시되고, 추가/수정/삭제 버튼이 있음
3. **Given** 관리자가 "새 사례 추가" 클릭, **When** 양식 표시, **Then** 작업 유형, 평수, 문제점, 장비, 방식, 결과, 피드백 입력 필드가 있음

---

### Edge Cases

- 모바일에서 대량의 텍스트(후기, 사례)를 어떻게 가독성 있게 표시할 것인가?
- 여러 언어를 지원해야 하는가? (현재는 한국어만 가정)
- SNS 링크가 변경되거나 없어질 경우 어떻게 처리할 것인가?
- 예약 폼(구글폼)이 변경되거나 새로운 폼이 필요할 경우 어떻게 동적으로 관리할 것인가?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 웹사이트는 반응형 디자인(모바일, 태블릿, 데스크톱)을 지원해야 함
- **FR-002**: 홈(Home) 페이지는 다음 섹션을 포함해야 함: 히어로 섹션(메인 카피, 서브 카피, 버튼 3개), 브랜드 철학(6개 항목), 핵심 서비스 카드(3개)
- **FR-003**: 소개(About) 페이지는 브랜드 소개, 브랜드 철학 6개 항목(상세)을 포함해야 함
- **FR-004**: 서비스(Service) 페이지는 서비스 인트로, 3개 서비스 카드(입주청소, 이사청소, 인테리어 후 청소) 상세 설명, 기본 청소 범위(7개 영역), 제한/옵션 안내를 포함해야 함
- **FR-005**: 작업 방식(How We Work) 페이지는 9단계 프로세스를 단계별로 시각화하고, 각 단계 설명과 주요 강점 3개(하루 한 집 전담, 전문 장비, 실시간 공유)를 포함해야 함
- **FR-006**: 장비 & 시스템(Equipment) 페이지는 4개 장비(전문 흡입 시스템, 고온 스팀 시스템, 바닥 폴리싱 시스템, 친환경 약품)와 각 장비 설명을 포함해야 함
- **FR-007**: 시공 사례(Case Study) 페이지는 최소 3개의 사례를 표시하고, 각 사례는 작업 유형, 평수, 현장 문제점, 사용 장비, 실제 작업 방식, 작업 결과, 고객 피드백 요약을 포함해야 함
- **FR-008**: 후기(Review) 페이지는 고객 후기를 표시하고, 최소 6개의 강조 메시지(차이 확실, 장비 수준, 전문성, 가성비, 투명성, 완성도)를 포함해야 함
- **FR-009**: 이용 안내(Notice) 페이지는 5개 섹션(작업 제한 및 안내, 추가 비용 기준, 기본 준비 사항, A/S 정책, 취소·위약금)을 포함하고, 각 섹션은 확장 가능해야 함(아코디언 또는 탭)
- **FR-010**: 네비게이션 메뉴는 데스크톱에서는 가로형 메뉴, 모바일에서는 햄버거 메뉴를 지원해야 함
- **FR-011**: 웹사이트의 모든 페이지 헤더에는 로고, 메뉴, "견적 / 예약" 버튼이 포함되어야 함
- **FR-012**: 웹사이트의 모든 페이지 푸터에는 회사 정보, 소셜 미디어 링크(유튜브, 인스타그램), 필수 안내 사항이 포함되어야 함
- **FR-013**: 홈 페이지 히어로 섹션에는 "간편견적문의" 버튼이 있어 구글폼으로 이동해야 함
- **FR-014**: 모든 서비스 및 사례 카드에는 "자세히 보기" 또는 "견적 / 예약 요청하기" 버튼이 있어 해당 상세 페이지 또는 구글폼으로 이동해야 함
- **FR-015**: 웹사이트의 색상 스킴은 블랙 베이스 + 블루 포인트를 사용해야 하고, 마블(대리석) 질감을 배경이나 섹션 구분에 활용해야 함
- **FR-016**: 웹사이트의 톤&무드는 고급, 단정, 모던, 절제된 느낌을 전달해야 함
- **FR-017**: 모든 텍스트 콘텐츠(메인 카피, 서브 카피, 설명, 안내)는 사용자 요청에 명시된 카피라이팅을 사용해야 함
- **FR-018**: 웹사이트는 서비스 제한 사항(부분 청소 불가, 시간 제한 청소 불가, 외창 청소 불가, 도배풀 미건조 상태 작업 불가, 실리콘 절개/재시공 불가, 공사 하자 제거 불가, 전기·수도 미사용 시 불가)을 명확하게 안내해야 함
- **FR-019**: 웹사이트는 추가 비용 발생 기준(곰팡이 심각, 니코틴 타르 변색, 스티커·끈끈이 대량, 준공청소 미진행, 평수·구성 차이, 엘리베이터 없음 등)을 명확하게 안내해야 함

### Key Entities *(include if feature involves data)*

- **Service**: 서비스 유형(입주청소, 이사청소, 인테리어 후 청소), 설명, 필요 상황, 기본 청소 범위
- **CaseStudy**: 작업 유형, 평수, 현장 문제점, 사용 장비, 실제 작업 방식, 작업 결과, 고객 피드백
- **Review**: 고객명(선택), 후기 텍스트, 별점(선택), 강조 메시지
- **ProcessStep**: 단계 번호, 단계명, 설명, 시각 자료(이미지/아이콘)
- **Equipment**: 장비명, 설명, 용도, 효과
- **BrandPhilosophy**: 철학 항목명, 설명, 아이콘

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 홈페이지에 진입하여 웹사이트의 목적과 브랜드 가치를 3초 이내에 이해할 수 있어야 함(메인 카피의 명확성)
- **SC-002**: 모든 주요 페이지(Home, About, Service, How We Work, Equipment, Case Study, Review, Notice, Reservation)가 완성되고, 각 페이지는 명시된 콘텐츠를 모두 포함해야 함
- **SC-003**: 모바일 디바이스에서 웹사이트의 모든 콘텐츠가 가독성 있게 표시되고, 햄버거 메뉴가 정상 작동해야 함
- **SC-004**: 사용자가 어느 페이지에서든 "견적 / 예약 요청하기" 버튼에 2회 클릭 이내로 도달할 수 있어야 함
- **SC-005**: 웹사이트의 모든 외부 링크(구글폼, 유튜브, 인스타그램, SNS)가 정상 작동해야 함
- **SC-006**: 웹사이트의 로딩 속도가 3초 이내여야 함(3G 네트워크 기준)
- **SC-007**: 웹사이트의 시각 디자인이 요청된 톤&무드(고급, 단정, 모던, 절제)와 색상(블랙 + 블루 + 마블 질감)을 일관되게 반영해야 함
- **SC-008**: 사용자가 "How We Work" 페이지를 보고, 회사의 투명한 작업 프로세스와 차별성(하루 한 집 전담, 실시간 촬영 공유)을 명확히 이해할 수 있어야 함
- **SC-009**: "Case Study" 페이지의 사례들이 실제 작업 과정과 결과를 명확히 보여주어, 사용자 신뢰도를 높여야 함
- **SC-010**: "Notice" 페이지에서 고객이 모든 제한 사항, 추가 비용, A/S 정책을 명확히 이해할 수 있어야 함(분쟁 감소)

## Assumptions

- 모든 외부 링크(구글폼, 유튜브, 인스타그램 URL)는 사용자가 제공하거나 사전 확인된 것으로 가정
- 시공 사례와 고객 후기는 실제 고객으로부터 제공되는 것으로 가정
- 웹사이트는 한국어를 기본 언어로 사용하며, 다국어 지원은 미래 기능으로 가정
- "마블 질감"은 배경 이미지, CSS 패턴, 또는 아이콘으로 표현 가능한 것으로 가정
- 모바일 반응형 디자인은 일반적인 모바일 화면 크기(320px ~ 1024px)를 기준으로 가정
- 견적/예약 폼의 관리는 구글폼으로 진행되며, 별도의 DB 연동은 미래 기능으로 가정
- 콘텐츠 관리 시스템(CMS) 구축은 초기 론칭 이후의 기능으로 가정
